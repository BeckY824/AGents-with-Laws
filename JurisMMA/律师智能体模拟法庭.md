# JurisMMA

多模态多智能体赋能的法律判决预测

## Abstract

法律判决预测（LJP Legal Judgement Prediction) : 基于事实描述预测法律案件的结果。

LJP框架，

- JurisMMA，有效地分解审判任务、规范流程并将其组织成不同的阶段。
- JurisMM1，包含10万条近期中国司法记录的大型数据集，其中包括文本和多模态视频文本数据。

## Introduction

法律语言具有高度专业化、复杂性强以及对精确术语依赖极高等特点，这给LJP带来了显著挑战。

- 简化真实法庭的模拟过程
- 使用RAG
- 优化提示工程

尽管NN分类器在简单预测任务重表现良好，但它们往往难以建模真实庭审中复杂的程序性流程与多方交互过程。

- 真实场景复现，多数框架无法：
  - 同时预测多个罪名/法律条款
  - 利用多模态证据（如庭审视频）导致预测准确性受限
- 多变量交互，现有方法的问题：
  - 角色模拟过于单一
  - 忽略不同角色之间的共享功能与流程设计
  - 结论片面&审判逻辑不合理

**JurisMMA** 一个多智能体法律判决预测框架，利用大语言模型的推理能力来模拟真实法庭过程。

![image-20260412162950542](/Users/edward_beck8n24/Library/Application Support/typora-user-images/image-20260412162950542.png)

包含两个核心模块：

1. 多智能体法庭模块，包含6个角色：初级法官，高级法官，首席法官，助手，被告，辩护律师。

   流程：案件记录 输入——> 初级法官提取案件信息 ——> 高级法官进行核查 ——> 生成案件总结——> 被告与律师参与 ——> 最终由首席法官裁决 ——> 如不满意，可上诉

2. 例，生成指控。（RAG+法律知识增强推理）

**数据集（JurisMM)** ，一个新数据集，超过10万条真实案件，源来自于官方，包含文本数据与多模态数据（83个视频-文本样本），解决了现有benchmark过时的问题。

## Method

六阶段审判流程

| **阶段**  | **角色**      |
| --------- | ------------- |
| 信息提取  | Junior Judge  |
| 校验      | Senior Judge  |
| 推理辅助  | Assistant     |
| 审理决策  | Chief Judge   |
| 辩护      | Legal Counsel |
| 陈述/上诉 | Defendant     |

> 这 6 个阶段本质上构成了一个 **“多智能体 + 检索增强 + 结构化决策 + 可上诉闭环” 的法律推理系统**。

## Experiments and results

为解决现有法律文本数据集（如 CAIL2018）存在的“内容过时”问题，我们从中国裁判文书网收集了 2014 年至 2024 年初共计 150,000 份公开刑事判决文书，覆盖所有类型的**刑事**案件。为确保数据符合《中华人民共和国刑法（2023 年修正案）》的规定，我们剔除了涉及 2014–2025 年期间修订条款的案件，并对类别分布进行了平衡处理。最终，构建了 JurisMM-Text 子集，共包含 101,544 个案件，其中：

- 94,494 条用于训练
- 7,050 条用于测试

为补充多模态法律数据，我们设计了一套半自动化流程来构建 JurisMM-Video 子集：

1. 首先，对视频进行分段处理，并利用视觉-语言模型筛选与犯罪现场、证据或嫌疑人行为相关的片段；
2. 随后，由法律专家对筛选结果进行人工审核，选取关键帧并设计问题；

最终，该流程生成了 83 个高质量的多模态样本，这些样本能够捕捉关键视觉信息，例如

- 证据
- 场景结构
- 嫌疑人行为

从而支持具备上下文理解能力的法律判决预测。

除了数据集之外，我们还构建了一个面向法律领域的知识库，该知识库基于 2023 年刑法修正案，包含三个核心组成部分：

- **法律条文（Legal Content）**：包含来自国家法律法规数据库（2023）的 438 条现行刑法条款，并辅以最高人民法院的司法解释；
- **罪名库（Charges）**：包含 483 类主要罪名的结构化列表，用于预测与检索；
- **判例库（Precedents）**：包含从 JurisMM-Text 中筛选出的 46,365 个代表性案例，用于辅助法律推理。

### Baselines

对于传统神经网络方法，我们选取了具有代表性且具有竞争力的基线模型，包括：

- TextCNN
- TOPJUDGE [4]
- MPBFN [5]

用于对我们的框架进行对比评估。尽管这些模型相对较早提出，但由于近年来在法律领域专用神经架构方面缺乏显著突破，它们在 JurisMM 数据集上仍然保持一定竞争力。

对于基于大语言模型（LLM）的基线方法，我们评估了以下开源模型：

- GLM-4V-9B [19]
- mPLUG-Owl-7B [20]
- Qwen2.5-VL-3B / 7B-Instruct [21]
- Qwen2.5-7B-Instruct [22]

以及闭源模型：

- GPT-4o-20250326 [23]

### Experiments setup

对于神经网络模型，我们在 JurisMM-Text 的训练集上进行训练，并在测试集上进行评估。训练过程中采用：

- Adam 优化器
- 学习率：10^{-3}
- Dropout 比例：0.5
- Batch size：512
- 训练轮数：16 个 epoch

对于大语言模型（LLMs）：

- **闭源模型**：将 temperature 固定为 0，以保证输出的确定性；
- **开源模型**：每个模型运行 3 次，并报告平均结果；

JurisMMA 框架主要使用 GPT-4o，同时还使用 Qwen2.5-VL-7B 进行了额外实验，以测试模型的泛化能力。每个输入平均消耗约 20,000 个 token（具体取决于案件复杂度）。

在性能评估方面，我们采用四种常用的多分类指标：

- 准确率（Accuracy, Acc.）
- 宏精度（Macro-Precision, MP）
- 宏召回率（Macro-Recall, MR）
- 宏 F1 值（Macro-F1, F1）

### Overall Performance 

三个任务难度排序：Law Articles法条预测（最容易） > Charges 罪名预测> Terms of Penalty 刑期预测（最难）

单纯的LLM不能做法律推理，结构才是关键。

- GPT-4o = 推理能力
- JurisMMA = 推理 + 结构 + 约束 + 交互 （多 agent 分工，RAG， 程序化流程）
